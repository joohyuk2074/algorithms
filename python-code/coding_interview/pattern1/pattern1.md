# Introduction to Two Pointers

## Two Pointers 패턴 소개

Two Pointers 패턴은 배열이나 문자열과 같은 **선형 자료구조(linear data structures)**를 효율적으로 탐색하기 위해 사용하는 알고리즘 기법이다.

이 패턴을 활용하면 다음과 같은 문제들을 효율적으로 해결할 수 있다.

- 팰린드롬(회문) 검사
- 배열 뒤집기
- 특정 합을 만족하는 두 원소 찾기

이 섹션에서는 다음을 이해하는 것이 목표이다.

- 언제 Two Pointers 패턴을 사용해야 하는지
- 포인터를 어떻게 이동시켜 문제를 효율적으로 해결하는지
- 코드 성능을 최적화하는 방법

---

# About the Pattern

## Two Pointers 패턴이란?

Two Pointers 패턴은 **배열이나 연결 리스트 같은 순차적인 데이터 구조를 효율적으로 탐색하거나 조작하기 위한 문제 해결 기법**이다.

이름 그대로 **두 개의 포인터(pointer)**를 사용한다.

이 두 포인터는 다음과 같은 방식으로 동작한다.

- 서로 다른 위치에서 시작
- 서로 반대 방향으로 이동
- 특정 조건에 따라 이동 방향이나 위치 변경

이렇게 두 포인터가 **협력(coordinated manner)**하여 데이터 구조를 탐색함으로써 효율적인 알고리즘을 만들 수 있다.

이 방식은 특히 **시간 복잡도와 공간 복잡도를 최적화**하는 데 유리하다.

---

# 언제 사용해야 하는가

배열에서 **특정 조건을 만족하는 두 개의 데이터를 찾는 문제**가 나오면  
Two Pointers 패턴을 먼저 떠올리는 것이 좋다.

예를 들면 다음과 같은 문제들이 있다.

- 합이 특정 값이 되는 두 숫자 찾기
- 문자열이 팰린드롬인지 확인
- 배열에서 조건을 만족하는 쌍(pair) 찾기

# 포인터 이동 방식

문제에 따라 포인터는 다음과 같은 방식으로 이동할 수 있다.

- 한 방향으로 함께 이동
- 서로 반대 방향으로 이동
- 조건에 따라 포인터 위치 조정

# 팰린드롬(Palindrome) 탐지

> **팰린드롬**이란 앞에서 읽어도, 뒤에서 읽어도 동일한 문자열을 말한다.  
> 주어진 문자열이 팰린드롬인지 판별하라.

---

## 핵심 아이디어 — Two Pointer (양방향 포인터)

두 개의 포인터(`p1`, `p2`)를 문자열의 **양 끝**에 초기화한 뒤,  
매 반복마다 두 포인터가 가리키는 문자를 **비교**하고,  
일치하면 포인터를 **안쪽으로** 이동시킨다.

---

## 단계별 시각화

### Step 1 — 초기 상태

문자열 `"madam"` 을 배열로 표현한다.

```
인덱스:  0    1    2    3    4
       +----+----+----+----+----+
       | m  | a  | d  | a  | m  |
       +----+----+----+----+----+
         ↑                   ↑
        p1                  p2
```

- `p1 = 0` (문자열의 시작)
- `p2 = 4` (문자열의 끝)

---

### Step 2 — 첫 번째 비교 및 이동

`p1`이 가리키는 `'m'`과 `p2`가 가리키는 `'m'`을 비교한다.

```
       +----+----+----+----+----+
       | m  | a  | d  | a  | m  |
       +----+----+----+----+----+
         ✅                  ✅
         ↑                   ↑
        p1                  p2

결과: 'm' == 'm' → 일치 ✅
```

두 문자가 일치하므로 포인터를 안쪽으로 이동한다.

```
p1: 0 → 1
p2: 4 → 3
```

---

### Step 3 — 두 번째 비교 및 이동

```
       +----+----+----+----+----+
       | m  | a  | d  | a  | m  |
       +----+----+----+----+----+
         ✅   ↑         ↑   ✅
             p1        p2

결과: 'a' == 'a' → 일치 ✅
```

두 문자가 일치하므로 포인터를 다시 안쪽으로 이동한다.

```
p1: 1 → 2
p2: 3 → 2
```

---

### Step 4 — 포인터 교차 (종료 조건)

```
       +----+----+----+----+----+
       | m  | a  | d  | a  | m  |
       +----+----+----+----+----+
         ✅   ✅   ↑↑   ✅   ✅
                 p1
                 p2

결과: p1 >= p2 → 반복 종료
```

`p1`과 `p2`가 동일한 위치(중앙)에서 만났다.  
모든 비교에서 불일치가 없었으므로 **팰린드롬으로 판별**한다.

---

### Step 5 — 최종 결론

```
       +----+----+----+----+----+
       | m  | a  | d  | a  | m  |
       +----+----+----+----+----+
         ✅   ✅   ✅   ✅   ✅

→ "madam" 은 팰린드롬이다. ✅
```

## 구현 코드

### Python

```python
def is_palindrome(s: str) -> bool:
    p1, p2 = 0, len(s) - 1
    while p1 < p2:
        if s[p1] != s[p2]:
            return False
        p1 += 1
        p2 -= 1
    return True

# 테스트
print(is_palindrome("madam"))   # True
print(is_palindrome("hello"))   # False
```

### Java

```java
public boolean isPalindrome(String s) {
    int p1 = 0, p2 = s.length() - 1;
    while (p1 < p2) {
        if (s.charAt(p1) != s.charAt(p2)) {
            return false;
        }
        p1++;
        p2--;
    }
    return true;
}
```

## 복잡도 분석

| 구분 | 복잡도 | 설명 |
|------|--------|------|
| 시간 복잡도 | `O(n)` | 문자열 길이의 절반만 순회 |
| 공간 복잡도 | `O(1)` | 포인터 2개만 사용, 추가 메모리 없음 |

---

## 핵심 포인트

- **Two Pointer** 패턴의 가장 기본적인 활용 사례
- 문자열을 뒤집어 비교하는 방식(`s == s[::-1]`)도 가능하지만, 이는 `O(n)` 추가 공간을 사용한다
- Two Pointer 방식은 **In-Place**로 동작하므로 공간 효율이 더 우수하다

## 실제 활용 사례

Two Pointers 패턴은 현실 세계의 다양한 문제에서 사용된다.

### 🖥️ 메모리 관리 (Memory Management)

Two Pointers 패턴은 **메모리 할당과 해제**에 있어 핵심적인 역할을 한다.

```
메모리 풀(Memory Pool) 초기 상태

+--------+--------+--------+--------+--------+--------+
|  사용  |  사용  | 가용   | 가용   | 가용   | 가용   |
| 중인   | 중인   | 메모리 | 메모리 | 메모리 | 메모리 |
+--------+--------+--------+--------+--------+--------+
                    ↑                            ↑
                  start                         end
                 pointer                       pointer
                (이동 가능)                  (고정, 경계 보호)
```

**동작 방식**

- **`start` 포인터**: 사용 가능한 메모리 블록의 **시작점**을 가리킨다.
- **`end` 포인터**: 메모리 블록의 **끝점**을 가리키며, 경계 역할을 한다.

**메모리 할당 시 (Allocation)**

```
프로세스가 메모리 요청
         │
         ▼
start 포인터를 앞으로 이동 (→)
         │
         ▼
새로운 메모리 블록을 해당 프로세스에 할당
```

**메모리 해제 시 (Deallocation)**

```
프로세스가 메모리 반환
         │
         ▼
start 포인터를 뒤로 이동 (←)
         │
         ▼
해제된 메모리를 다시 "가용" 상태로 표시
```

> **핵심:** `end` 포인터는 항상 **고정**된 상태를 유지하여  
> 메모리 할당이 서로 겹치는(overlapping) 오류를 방지하는 **안전 장치** 역할을 한다.
